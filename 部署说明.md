# 博客部署说明

## GitHub Pages 访问地址

你的博客静态网页访问地址为:
**https://60777flowers.github.io/blog-refactor/**

或者等部署完成后,可以在以下位置查看:
1. 访问 https://github.com/60777flowers/blog-refactor
2. 点击上方的 "Settings" 标签
3. 在左侧菜单中找到 "Pages"
4. 在 "GitHub Pages" 部分会显示访问地址

## 如何启用 GitHub Pages

我已经创建了 GitHub Actions 工作流,会自动部署你的博客。请按以下步骤操作:

### 步骤 1: 启用 GitHub Pages

1. 访问 https://github.com/60777flowers/blog-refactor/settings/pages
2. 在 "Build and deployment" 部分:
   - Source: 选择 **Deploy from a branch**
   - Branch: 选择 **main** 和 **/ (root)**
   - 点击 "Save"

### 步骤 2: 等待部署完成

1. 刚打开的页面应该会显示 "Your site is live at https://60777flowers.github.io/blog-refactor/"
2. 如果显示 "Your site is being deployed",请等待 2-5 分钟
3. 刷新页面,直到显示 "Your site is live at..."

### 步骤 3: 访问博客

点击显示的链接即可访问你的博客!

## 自动部署

## 故障排除

### 如果显示 "404 Not Found"

1. 等待 2-5 分钟,部署可能还在进行中
2. 检查 Actions 页面: https://github.com/60777flowers/blog-refactor/actions
3. 如果 Actions 失败,查看错误日志并修复

### 如果 Actions 失败

1. 访问 https://github.com/60777flowers/blog-refactor/actions
2. 点击失败的 workflow 查看详细错误
3. 常见问题:
   - 依赖安装失败 → 检查 package.json
   - 主题克隆失败 → 检查网络连接
   - 生成失败 → 检查文章格式

## 首次部署

由于这是首次部署,可能需要:
1. 在 GitHub 仓库中启用 Pages 功能
2. 等待几分钟让 GitHub 完成初始化
3. 刷新 Pages 设置页面确认部署状态
