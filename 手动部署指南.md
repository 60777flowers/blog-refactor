# 手动部署博客到 GitHub Pages

## 当前问题

GitHub Actions 自动部署失败,网络连接不稳定。我们改用更简单可靠的手动部署方式。

## 方法一:使用 Hexo 内置的 Git 部署器(推荐)

### 步骤 1: 修改 Hexo 配置

编辑 `blog/_config.yml`,找到 `deploy` 部分,修改为:

```yaml
deploy:
  type: git
  repo: https://60777flowers:Qazxswedc@12321@github.com/60777flowers/60777flowers.github.io.git
  branch: main
  message: "Site updated: {{ now('YYYY-MM-DD HH:mm:ss') }}"
```

**注意**: 你需要先创建一个名为 `60777flowers.github.io` 的 GitHub 仓库。

### 步骤 2: 安装部署插件

```bash
cd d:\大四下\博客修复与重整\blog
npm install hexo-deployer-git --save
```

### 步骤 3: 克隆 Butterfly 主题

```bash
cd d:\大四下\博客修复与重整\blog\themes
git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git butterfly
```

### 步骤 4: 生成并部署

```bash
cd d:\大四下\博客修复与重整\blog
npx hexo clean
npx hexo generate
npx hexo deploy
```

### 步骤 5: 启用 GitHub Pages

1. 访问 https://github.com/60777flowers/60777flowers.github.io/settings/pages
2. Source 选择 **Deploy from a branch**
3. Branch 选择 **main** 和 **/(root)**
4. 点击 Save

### 访问博客

**https://60777flowers.github.io/**

## 方法二:手动上传 public 目录(备用方案)

如果方法一不行,可以手动上传:

### 步骤 1: 生成静态文件

```bash
cd d:\大四下\博客修复与重整\blog
npx hexo clean
npx hexo generate
```

### 步骤 2: 创建 GitHub Pages 仓库

1. 访问 https://github.com/new
2. Repository name: `60777flowers.github.io`
3. Public/Private: Public
4. 点击 Create

### 步骤 3: 克隆仓库

```bash
cd d:\大四下\博客修复与重整
git clone https://60777flowers:Qazxswedc@12321@github.com/60777flowers/60777flowers.github.io.git
```

### 步骤 4: 复制文件

```bash
cd 60777flowers.github.io
xcopy ..\blog\public\*.* /E /Y
```

### 步骤 5: 推送到 GitHub

```bash
cd d:\大四下\博客修复与重整\60777flowers.github.io
git add .
git commit -m "update blog"
git push origin main
```

### 启用 GitHub Pages

访问 https://github.com/60777flowers/60777flowers.github.io/settings/pages,按照方法一的步骤5启用。

## 推荐使用方法一

方法一(使用 Hexo deployer)的优势:
- ✅ 自动化程度高
- ✅ 配置一次后,之后只需一个命令
- ✅ 支持自定义提交信息
- ✅ 更稳定可靠
